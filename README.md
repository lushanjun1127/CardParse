## 一、项目概述

本项目是一个**零依赖、全本地化**的 VCF联系人批量生成工具，通过纯前端技术实现从文本解析到标准VCF文件生成的完整流程。

### 核心理念
- ✅ 隐私优先 - 所有数据在浏览器端完成，永不上传
- ✅ 零部署成本 - 单 HTML 文件即可运行
- ✅ 极致简洁 - 原生 JavaScript + CSS/JS分离

---

## 二、第一性原理分析

问题：为何要做VCF生成器？    解决方案：解决联系人批量导入手机通讯录的需求痛点

问题：为何选纯前端方案？   解决方案：无服务器请求=零数据泄露风险+零运维成本

问题：支持哪些格式？       解决方案：中文/英文/字母数字组合混合姓名

问题：如何实现快速生成？   解决方案：文本流解析 + 正则表达式匹配

## 五、核心功能

【基础功能】
✨ 简单易用 - 直接粘贴联系人信息即可生成VCF文件
📱 响应式设计 - 适配桌面和移动设备

## 六、输入格式规范

支持的联系人文本格式:

张三 13812345678
李四 13987654321
13712345678 王五
A0397 13612345679

规则说明:
- 姓名与电话号码之间用空格或制表符分隔
- 顺序可互换: [姓名][电话] 或 [电话][姓名]
- 支持中文名/英文名/字母数字组合

---

## 七、操作步骤流程

步骤1: 在左侧文本框中输入联系人信息 (姓名和电话之间用空格或制表符分隔)

步骤2: 点击"解析数据"按钮进行数据解析与验证

步骤3: 在中间面板查看联系人预览和统计信息

步骤4: 点击"下载VCF"保存文件 或 点击"复制"按钮复制VCF内容

步骤5: 可使用"示例"按钮快速填充示例数据测试

步骤6: 可将VCF内容粘贴到右侧区域，点击"解析VCF"反向解析为联系人列表

## 八、安全性矩阵

风险等级: 高 | 威胁面: 数据泄露 | 缓解措施: 所有数据处理均在客户端本地完成

风险等级: 中 | 威胁面: XSS注入 | 缓解措施: 对用户输入进行了验证和清理

风险等级: 低 | 威胁面: 误解析 | 缓解措施: 正则表达式严格匹配电话号码格式

风险等级: 低 | 威胁面: 跨站脚本 | 缓解措施: 实现了XSS防护机制
